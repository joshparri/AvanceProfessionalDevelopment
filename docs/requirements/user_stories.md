# User Stories and Acceptance Criteria

## Dashboard
### Story
As Josh, I want to open the app and immediately understand what matters today.

### Acceptance criteria
- dashboard loads with today / next shift visible
- open tasks are shown by urgency
- recent handover is visible
- time logged this invoice cycle is visible
- at least 3 quick actions are available

## Shift prep
### Story
As Josh, I want a simple pre-shift routine so I do not arrive scattered.

### Acceptance criteria
- recurring Monday/Wednesday shifts can be created
- each shift can have a prep checklist
- each shift can store 3 priorities
- prior handover is visible from the shift view

## Work logging
### Story
As Josh, I want to capture what happened in under 30 seconds.

### Acceptance criteria
- quick-add work log form exists
- can link a work log to a client and task
- can record outcome and next step
- recent logs appear on dashboard

## Tasks
### Story
As Josh, I want follow-ups to survive the gap between shifts.

### Acceptance criteria
- tasks can be open, in progress, blocked, done
- tasks can have due dates
- overdue tasks are visually clear
- tasks can be linked to clients and work logs

## Knowledge
### Story
As Josh, I want to turn messy notes into useful future knowledge.

### Acceptance criteria
- knowledge entries support title, body, tags, confidence, last verified
- entries can be marked draft or trusted
- entries are searchable
- entries can link to clients and playbooks

## Playbooks
### Story
As Josh, I want repeat issues to become faster next time.

### Acceptance criteria
- playbooks can store step-by-step triage
- can log when a playbook was used
- can attach notes about what actually worked
- playbooks are searchable and filterable

## Time logging
### Story
As Josh, I want to track time and invoice cleanly.

### Acceptance criteria
- recurring shifts appear automatically
- actual hours can be edited
- hourly rate can be stored
- invoice preview can be generated from unbilled shifts
- shifts can be marked billed and paid

## Learning tracker
### Story
As Josh, I want my confidence and skills in IT MSP to grow intentionally through professional development.

### Acceptance criteria
- learning items can be created with topic and confidence score
- "ask team" items can be flagged
- "seen in real work" can be toggled
- next review date can be set
- progress in professional development can be tracked over time

## Security
### Story
As Josh, I want the app to be useful without becoming risky.

### Acceptance criteria
- app includes warning not to store passwords
- export/import exists
- local-first mode works
- any sync is optional
