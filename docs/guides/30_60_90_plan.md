# 30 / 60 / 90 Day Plan for Using the App

## First 30 days
Focus:
- capture everything
- build shift habit
- log time accurately
- create rough work notes (no private client details)
- save every recurring issue as a draft note
- start tracking professional development in IT MSP skills

Do not obsess over polish.
The win is consistency.

### App behaviour emphasis
- dashboard
- quick capture
- work logs
- tasks
- time logging

## Days 31–60
Focus:
- turn draft notes into trusted knowledge
- build 10–20 genuinely useful playbooks
- identify recurring client patterns
- improve invoice and handover workflow
- actively build IT MSP skills through learning tracker

### App behaviour emphasis
- knowledge verification
- linked notes
- playbook refinement
- better search hygiene
- professional development tracking

## Days 61–90
Focus:
- review what actually saves time
- remove friction
- improve templates
- add optional sync/export needs
- build personal strengths map
- assess professional development progress and plan next steps

### App behaviour emphasis
- playbook quality
- search speed
- reporting views
- reusable templates
- learning tracker discipline
- PD goal achievement
