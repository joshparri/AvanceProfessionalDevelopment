# Product Vision — Avance Work Companion

## One-line vision
A calm, private work companion that helps Josh step into Avance shifts prepared, think clearly under pressure, capture knowledge fast, build strong MSP habits, and grow his overall IT MSP skills over time.

## Problem
Josh is starting part-time MSP work only two days per week. That means:
- context can go stale between shifts
- follow-ups can fall through the cracks
- notes can get scattered
- repeat problems can feel “new” every time
- learning can stay vague instead of compounding
- invoicing and time logging can become annoying admin
- professional development in IT MSP skills needs intentional tracking

MSP work also tends to be:
- interrupt-driven
- context-switch heavy
- partly reactive
- partly preventative
- full of tiny details that matter later

The right app should reduce that friction and support skill growth.

## User
Primary user: Josh only.

He needs an app that supports:
- practical IT work
- light contractor admin
- technical confidence building
- fast re-entry after days away
- low mental overhead
- professional development and skill growth in IT MSP

## Outcomes
The app succeeds if it helps Josh:
1. start each shift with clarity
2. remember what matters from the previous shift
3. turn loose notes into trusted knowledge
4. resolve recurring issues faster
5. reduce avoidable stress and forgotten follow-ups
6. keep clean records of hours and billable work
7. steadily ramp up competence across tools, clients, and workflows
8. track and advance professional development in IT MSP skills

## Product principles
### 1. Re-entry first
Because Josh is only there Mondays and Wednesdays, the app must make “what was happening?” obvious within 60 seconds.

### 2. Capture under pressure
When work is busy, the app should make it easy to capture partial notes quickly and clean them up later.

### 3. Personal operator, not enterprise software
This app is for one person doing real work, not for impressing stakeholders with fake complexity.

### 4. Calm over flashy
The UI should help Josh breathe, focus, and act.

### 5. Trust by structure
Mark what is:
- known
- assumed
- unverified
- tested in real work

### 6. Compound learning
Every ticket, shift, and fix should be able to become future leverage.

### 7. Professional development focus
The app should actively support Josh's growth in IT MSP skills through structured tracking, reflection, and learning activities.

## Non-goals
This app is not trying to be:
- a full PSA
- an RMM
- a shared password vault
- a ticketing system for Avance clients
- a compliance platform
- an external client portal
- a generic all-in-one life app

## MVP
The MVP should focus on:
- dashboard
- shift prep
- work log / handover
- tasks / ticket capture
- knowledge base
- playbooks
- time logging and invoice support
- learning tracker
- search

## Later expansion
Possible later additions:
- optional Supabase sync
- email-to-note ingestion
- calendar sync
- template-based invoice export
- file attachments
- improved mobile quick capture
- smarter suggestion engine for repeated issues
