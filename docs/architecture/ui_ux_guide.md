# UI / UX and Tone Guide

## Desired feel
- quiet confidence
- clean and legible
- not sterile
- not corporate-bloated
- useful at 8:20am when the brain is not fully online
- useful at 4:55pm when tired

## Visual direction
- dark mode default okay
- soft contrast, not neon
- strong typography hierarchy
- spacious cards
- very clear buttons
- consistent icons
- obvious status chips
- strong search bar

## Interaction rules
- every core action should be within 1–2 clicks
- support keyboard shortcut for quick capture
- favour drawers and modals sparingly
- forms should autosave drafts where safe
- empty states should teach

## Tone of copy
Use copy like:
- “Next shift”
- “Pick up where you left off”
- “Capture it now, tidy later”
- “Needs follow-up”
- “Trusted note”
- “Draft note”
- “Needs verification”

Avoid copy like:
- “Leverage your workflow”
- “Supercharge productivity”
- “Revolutionise your IT ops”

## Stress-state UX
During busy work, the app should help the user:
- log something partial
- find the last relevant thing
- not lose context
- not overthink the structure

## Colour semantics
- red: urgent / blocked / overdue
- amber: needs review / unverified
- green: done / trusted / billed
- blue: informative / scheduled / linked
